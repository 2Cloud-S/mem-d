# Mem-D V0.7 — Recommendation Simulation Architecture

Status: Design (hardened, pre-implementation)

Date: 2026-06-17 (hardened 2026-06-17 per `V0.7-SIMULATION-ARCHITECTURE-AUDIT.md`)

Scope: architecture only. No memory mutation, no persistence, no workflow engine, no external systems, no execution.

---

# Purpose

Mem-D V0.6 established **recommendation generation** (ADR-001) and **recommendation evaluation** (ADR-002). The system can now answer:

- What is inside memory?
- What governance action should be considered per memory?

It cannot yet answer:

- **What would happen to the memory store if those recommendations were applied?**

That gap sits between recommendation quality and future workflow/action layers. Operators and downstream systems need dry-run impact estimates before queuing work, approving automation, or delegating to external memory stores.

The **Recommendation Simulation Layer** exists to close that gap **without executing anything**.

```
Diagnostics
    → Recommendations (V0.6)
    → Recommendation Evaluation (V0.6)
    → Simulation (V0.7)          ← this document
    → Workflows (V0.8, future)
    → Actions (V1.0, future)
```

Simulation is the first layer that models **post-recommendation store shape**. It remains read-only with respect to source data: all transformations occur on an in-memory copy that is discarded when the simulation completes.

Simulation does **not** validate whether recommendations are correct (ADR-002). It assumes `MemoryResolution` outputs are the simulation input contract and measures **operational impact** if those resolutions were applied under documented simulation semantics.

## Metric philosophy (normative)

All simulation metrics are **structural estimates** computed by adjusting pre-analysis cluster and resolution data against the simulated active store. They are **not** benchmark-equivalent measurements.

Simulation metrics must **not** be compared directly to, or presented as equivalent to:

- LongMemEval or PERMA baseline compression figures
- Clustering evaluation precision/recall
- Lifecycle or evolution gold-fixture accuracy
- Recommendation evaluation accuracy (ADR-002)

`AnalysisReport.metrics` values may be cited as **reference context only** (labeled `reference*` where used). Post-simulation values use the `estimated*` prefix to signal structural projection semantics.

# Scope

## What simulation may do

- Accept an analyzed memory store plus V0.6 recommendation outputs.
- Create a **deep, in-memory copy** of the source memory store.
- Apply **simulated transformations** derived from `MemoryResolution` records and supporting `Recommendation` metadata.
- Produce **estimated structural before/after metrics** comparing source store state to simulated store state (not benchmark-equivalent compression).
- Emit a structured **SimulationReport** with traceability back to recommendation IDs, roles, and evidence references.
- Support selective simulation modes (e.g. simulate only `merge` + `archive`, or full resolution set).
- Run deterministically: same inputs → same simulated store → same metrics.

## What simulation must never do

| Prohibited behavior | Rationale |
| --- | --- |
| Modify source `MemoryRecord` objects or input files | Mem-D analyzes; it does not manage memory |
| Persist simulated stores to disk or external systems | No persistence layer in V0.7 |
| Write back to agent memory providers | Execution is out of scope |
| Re-plan or override recommendations | Simulation consumes recommendations; ADR-001 outputs remain immutable. Orphan-merge safety adjusts **simulation accounting only**, not resolutions |
| Invoke workflow engines, task queues, or automation | Workflows are V0.8+ |
| Call external APIs or MCP servers | Local-first, self-contained |
| Perform semantic merge of content (LLM consolidation) | Content fusion is an execution concern; simulation models structural removal/retention only |
| Re-run full analysis pipeline on simulated store (V0.7 default) | Optional future extension; not required for initial impact metrics |

Simulation answers **"what changes in store composition?"**, not **"what should the merged text say?"**

# Inputs

Simulation consumes outputs already produced by `analyze_file()` (or an equivalent in-memory `AnalysisReport`). No new inference is required.

## Memory store

- **Source:** `AnalysisReport.memories` (`tuple[MemoryRecord, ...]`)
- Each record: `id`, `content`, optional metadata (`source`, timestamps, tags)
- The simulation layer treats this tuple as the authoritative pre-simulation store.

## Recommendations

- **Source:** `AnalysisReport.recommendations` (`tuple[Recommendation, ...]`)
- Group-scoped records with `recommendationId`, `action`, `affected_memories`, `evidence`, `estimatedImpact`
- Used for explainability, role assignment (keeper/removable/archive_candidate), and utilization accounting.
- Not the sole driver of transformation — per-memory resolutions are authoritative.

## Memory resolutions

- **Source:** `AnalysisReport.memoryResolutions` (`tuple[MemoryResolution, ...]`)
- **Primary simulation driver:** one `resolvedAction` per `memoryId`
- Fields used:
  - `resolvedAction` — which simulation handler to apply
  - `role` — `keeper`, `removable`, `archive_candidate`, `successor`, `review`, `retain`
  - `recommendationId` — traceability
  - `confidence`, `conflictDetected`, `suppressedActions` — explainability and review accounting

## Lifecycle states

- **Source:** `AnalysisReport.validation["memoryLifecycle"]["memoryLifecycleAssignments"]`
- Used for:
  - before/after lifecycle distribution metrics
  - archive simulation context (state labels on removed records)
  - explainability links to lifecycle evidence
- Simulation does **not** re-infer lifecycle; it consumes pre-computed assignments.

## Governance actions

- **Source:** `AnalysisReport.actions` (`tuple[GovernanceAction, ...]`)
- Used for:
  - trust and policy context in explainability (`evidenceRefs` enrichment only)
  - cross-reference to cluster-level merge drivers (`clusterId`, `actionId`)
- Simulation does **not** re-apply policy and does **not** compute utilization from governance-action joins. Utilization is derived from **recommendation outcomes** (`MemoryResolution` + simulated event logs) only.

## Trust information

- **Source:** `AnalysisReport.clusters` (`trustScore`, `trustLevel`, `trustReasons`) and `validation.clusterQuality`
- Used for:
  - estimated trusted structural compression calculations (HIGH-trust cluster membership only)
  - explainability (why a merge simulation counted as trusted)
  - optional filtering (e.g. simulate only resolutions backed by HIGH-trust clusters)

# Simulation Actions

Simulation applies one handler per `MemoryResolution`. Handlers operate on the **simulated store copy** only.

## `merge`

**Meaning in simulation:** duplicate or preference-cluster consolidation is modeled as **removal of non-keeper members** from the active simulated store. No content fusion occurs.

| Role | Simulated effect |
| --- | --- |
| `keeper` | Record remains in active simulated store |
| `removable` | Record removed from active simulated store; logged in `simulatedMerges` |

**Rules:**

1. Only memories with `resolvedAction == merge` participate in merge simulation.
2. Removable records (`role == removable`) are excluded from the active simulated store and from estimated structural duplicate counts.
3. Keeper records (`role == keeper`) remain in the active simulated store; content is unchanged.
4. Merge groups are reconstructed from `Recommendation.affected_memories` or by grouping resolutions sharing the same `recommendationId`.
5. Orphan-merge safety applies when a merge group lacks a valid keeper in the active store — see [Orphan merge safety semantics](#orphan-merge-safety-semantics). This is simulation-only behavior; original `MemoryResolution` and `Recommendation` outputs are preserved unchanged in the input `AnalysisReport`.

**What merge does NOT mean:** generating a new consolidated memory body, updating embeddings, or deleting data from source files.

## `archive`

**Meaning in simulation:** lifecycle-eligible retirement is modeled as **removal from the active simulated store** with archival metadata preserved in simulation logs.

| Role | Simulated effect |
| --- | --- |
| `archive_candidate` | Removed from active simulated store; entry added to `simulatedArchives` |
| `successor` | Remains in active store (often paired with `keep` resolution) |

**Rules:**

1. Only `resolvedAction == archive` triggers archival removal.
2. Archived records retain their original `MemoryRecord` payload in the archive log for traceability.
3. Lifecycle state at time of simulation is captured in archive log entries.
4. Archive simulation does not model cold-storage tiers, TTL, or provider-specific retirement APIs.

## `review`

**Meaning in simulation:** human judgment is required; the record **stays in the active simulated store unchanged**.

| Role | Simulated effect |
| --- | --- |
| `review` | No structural change; counted in `unresolvedReviewCount` |

**Rules:**

1. Review resolutions never remove or merge records in V0.7 simulation.
2. Records with `conflictDetected == true` are always counted as unresolved review even if a resolution exists.
3. Review items appear in `simulatedReviewQueue` for operator-facing output.
4. Optional `simulateReviewAsNoOp` is the only supported mode in V0.7 (no "assume approved merge" branching — that would be a workflow concern).

## `keep`

**Meaning in simulation:** explicit or implicit retention; record **remains in active simulated store unchanged**.

| Role | Simulated effect |
| --- | --- |
| `retain`, `keeper` (when not merge-removal), empty role with `keep` | No change |

**Rules:**

1. Default for memories without a remediation resolution.
2. Keep does not add records; it preserves status quo in the simulated copy.
3. Keep resolutions are counted for utilization denominators but produce no store delta.

## `defer` (internal)

- V0.6 `defer` recommendations are **excluded** from simulation by default.
- If present in resolutions, treated as `keep` (no simulated change) unless a future flag explicitly includes deferred actions.

# Orphan merge safety semantics

When a merge group references memories whose **keeper is absent from the active simulated store** (never existed, already removed by a prior simulated archive step, or missing from `recommendationId` grouping), simulation applies a **simulation-only safety guard**. This does **not** modify, re-plan, or overwrite V0.6 recommendation outputs.

## Trigger conditions

Orphan-merge handling activates when **all** of the following hold:

1. One or more memories in a merge group have `resolvedAction == merge` and `role == removable`.
2. No memory in the same merge group has `role == keeper` **and** is present in the active simulated store at merge-processing time.
3. Simulation mode includes merge processing (`full` or `merge_only`).

## Downgrade behavior (simulation-only)

| Aspect | Behavior |
| --- | --- |
| Structural effect | **No removal** — all group members remain in the active simulated store |
| Simulated classification | Affected removables are treated as **unresolved review** for simulation accounting (`unresolvedReviewCount`, `simulatedReviewQueue`) |
| Original recommendations | `AnalysisReport.recommendations` and `AnalysisReport.memoryResolutions` are **unchanged** — still reflect ADR-001 output |
| Merge log | **No** `SimulatedMergeGroup` entry is emitted for the failed group |

## Warning generation

Emit one `SimulationWarning` per affected merge group:

```
SimulationWarning:
  code: "ORPHAN_MERGE_NO_KEEPER"
  memoryId: <first removable memoryId in group>
  recommendationId: <group recommendationId>
  message: "Merge simulation skipped: no keeper present in active store for recommendation group."
```

Warnings are included in `SimulationReport.simulationWarnings` and increment `simulationWarningCount`.

## Preservation of original recommendation outputs

Simulation **never writes back** to `AnalysisReport`. Orphan-merge downgrade affects only:

- the in-memory `simulated_store` copy
- `SimulationReport` event logs and metrics
- `simulatedReviewQueue` entries derived from the safety guard

Downstream consumers comparing recommendations vs simulation must treat orphan cases as **"resolution said merge; simulation refused removal for safety"**, not as a recommendation correction.

# Store Transformation Model

## Simulated store creation

```
source_store = AnalysisReport.memories
simulated_store = deep_copy(source_store)   # new list/tuple of new MemoryRecord instances
archive_log = []
merge_log = []
review_queue = []
```

**Invariants at creation:**

- `len(simulated_store) == len(source_store)` before any handler runs.
- Source `AnalysisReport` and original `MemoryRecord` objects are never mutated.
- Deep copy applies to record containers and record objects; shared references inside `validation` dicts are not modified.

## Application order

To avoid double-processing and role conflicts, simulation applies handlers in a fixed order:

```
1. Build resolution index: memoryId → MemoryResolution
2. Build recommendation index: recommendationId → Recommendation (may be partial)
3. Process merge removals (group by recommendationId; apply orphan-merge safety)
4. Process archive removals (remove archive_candidates not already removed)
5. Classify review and keep (no structural change)
6. Compute estimated structural metrics on source vs simulated active store
```

**Precedence rationale:** aligns with ADR-001 safety ordering. Archive and merge should not target the same memory (conflicts resolve to `review`). If an edge case occurs, simulation emits a warning and skips removal.

## Merge candidate representation

Each simulated merge is logged as:

```
SimulatedMergeGroup:
  recommendationId: str
  keeperId: str
  removedIds: tuple[str, ...]
  clusterId: str | None          # from governance action target when available
  trustLevel: str | None         # High | Medium | Low
  trusted: bool                  # derived from cluster trust
  evidenceRefs: tuple[str, ...]  # canonical refs; see Explainability → evidenceRefs encoding
```

The active `simulated_store` no longer contains `removedIds`. The keeper record is unchanged.

## Archive representation

Each simulated archive is logged as:

```
SimulatedArchiveEntry:
  memoryId: str
  lifecycleState: str
  recommendationId: str
  archivedRecord: MemoryRecord    # snapshot at simulation time
  evidenceRefs: tuple[str, ...]
```

Archived records are removed from active `simulated_store` but preserved in `archive_log` inside the report.

## Review representation

```
SimulatedReviewEntry:
  memoryId: str
  recommendationId: str
  reason: str                      # from Recommendation.reason or explainability fallback
  conflictDetected: bool
  suppressedActions: tuple[str, ...]
  evidenceRefs: tuple[str, ...]
```

Review records remain in active `simulated_store`.

## Active vs removed accounting

After simulation:

| Set | Definition |
| --- | --- |
| Active simulated memories | Records still in `simulated_store` after merge/archive removals |
| Simulated removals | Union of merge removables and archive candidates successfully removed |
| Unchanged | Records with `keep` or `review` resolutions |

# Metrics

Simulation produces **estimated structural deltas** and **utilization counters**. Metrics are computed from store composition, source cluster membership maps, and resolution outcomes — **not** from re-clustering, re-embedding, or re-running `calculate_metrics()` unless explicitly added in a future mode.

All percentage fields in this section are **structural estimates**. Report surfaces must label them as simulated structural estimates, not analyze or benchmark compression figures.

## Normative helper definitions

Let:

- `S_before` = set of memory IDs in source store
- `S_after` = set of memory IDs in active simulated store after handlers complete
- `C` = source `AnalysisReport.clusters` (membership maps from pre-simulation analysis)
- `N(x, y)` = safe percentage: `(x / y) * 100` when `y > 0`, else `0.0`

**Per-cluster removable estimate** (structural, not re-clustered):

```
active_members(cluster, S) = { m in cluster.members | m in S }
removable_estimate(cluster, S) = max(0, len(active_members(cluster, S)) - 1)
```

**HIGH-trust removable estimate** (structural):

```
trusted_removable_estimate(cluster, S) =
  removable_estimate(cluster, S)  if cluster.trustLevel == High
  else 0
```

## Core metric definitions

### Memory count delta

| Field | Exact definition |
| --- | --- |
| `memoryCountBefore` | `len(S_before)` |
| `memoryCountAfter` | `len(S_after)` |
| `memoryCountDelta` | `memoryCountAfter - memoryCountBefore` (negative = reduction) |
| `memoryReductionPercentage` | `N(abs(memoryCountDelta), memoryCountBefore)` when `memoryCountDelta < 0`, else `0.0` |

### Estimated duplicate reduction

| Field | Exact definition |
| --- | --- |
| `estimatedRemovableBefore` | `sum(removable_estimate(c, S_before) for c in C)` |
| `estimatedRemovableAfter` | `sum(removable_estimate(c, S_after) for c in C)` |
| `estimatedDuplicateReduction` | `estimatedRemovableBefore - estimatedRemovableAfter` |

This aligns with the **removable-slot** interpretation used by `memd/metrics.py` (`duplicate_count = |duplicate_members| - |clusters|`), but is recomputed on `S_after` using **source** cluster maps only.

### Estimated compression gain (structural)

| Field | Exact definition |
| --- | --- |
| `estimatedStructuralCompressionBefore` | `N(estimatedRemovableBefore, memoryCountBefore)` |
| `estimatedStructuralCompressionAfter` | `N(estimatedRemovableAfter, memoryCountAfter)` |
| `estimatedCompressionGain` | `estimatedStructuralCompressionBefore - estimatedStructuralCompressionAfter` (positive = estimated improvement) |
| `estimatedTrustedRemovableBefore` | `sum(trusted_removable_estimate(c, S_before) for c in C)` |
| `estimatedTrustedRemovableAfter` | `sum(trusted_removable_estimate(c, S_after) for c in C)` |
| `estimatedTrustedStructuralCompressionBefore` | `N(estimatedTrustedRemovableBefore, memoryCountBefore)` |
| `estimatedTrustedStructuralCompressionAfter` | `N(estimatedTrustedRemovableAfter, memoryCountAfter)` |
| `estimatedTrustedCompressionGain` | `estimatedTrustedStructuralCompressionBefore - estimatedTrustedStructuralCompressionAfter` |

**Reference context only (not simulation output equivalents):**

| Field | Source | Usage |
| --- | --- | --- |
| `referenceCompressionOpportunity` | `AnalysisReport.metrics.compressionOpportunity` | Pre-simulation analyze context in reports; do not compare numerically to `estimatedStructuralCompressionAfter` as if re-analyzed |
| `referenceTrustedCompressionOpportunity` | `AnalysisReport.metrics.trustedCompressionOpportunity` | Same |

### Lifecycle distribution change

Lifecycle metrics use `memoryLifecycleAssignments` keyed by memory ID.

| Field | Exact definition |
| --- | --- |
| `lifecycleDistributionBefore` | For each lifecycle state `L`: count of assignments where `memoryId in S_before` and `lifecycleState == L` |
| `lifecycleDistributionAfter` | For each lifecycle state `L`: count of assignments where `memoryId in S_after` and `lifecycleState == L` |
| `lifecycleDistributionChange` | Per-state: `lifecycleDistributionAfter[L] - lifecycleDistributionBefore[L]` |
| `archivedByLifecycleState` | For each `SimulatedArchiveEntry`: increment count for `lifecycleState` (simulation log breakdown; independent of active distribution) |

**Normative rule:** Removed memories are **excluded** from `lifecycleDistributionAfter` counts. They are **not** reclassified into a synthetic `Archived` lifecycle bucket in V0.7. Archive effect is visible in `archivedByLifecycleState` and `simulatedArchives`.

### Recommendation utilization

Utilization is computed from **recommendation outcomes** (`MemoryResolution` + simulated event logs), not governance-action joins.

| Field | Exact definition |
| --- | --- |
| `totalResolutions` | `len(memoryResolutions)` |
| `resolutionsApplied` | Count of `memoryId` where ID ∈ `S_before` and ID ∉ `S_after` (structural removal occurred) |
| `resolutionsNoOp` | `totalResolutions - resolutionsApplied` |
| `mergeGroupsSimulated` | `len(simulatedMerges)` (successful groups only; orphan-merge groups excluded) |
| `archivesSimulated` | `len(simulatedArchives)` |
| `recommendationUtilizationRate` | `resolutionsApplied / totalResolutions` when `totalResolutions > 0`, else `0.0` |
| `recommendationsEligible` | Distinct `recommendationId` from resolutions where `resolvedAction in {merge, archive}` and simulation mode includes that action |
| `recommendationsWithStructuralEffect` | Distinct `recommendationId` appearing in `simulatedMerges` or `simulatedArchives` |
| `recommendationOutcomeUtilizationRate` | `recommendationsWithStructuralEffect / recommendationsEligible` when `recommendationsEligible > 0`, else `0.0` |

**Note:** `recommendationUtilizationRate` is per-memory; `recommendationOutcomeUtilizationRate` is per recommendation group. Both are operational diagnostics, not quality scores.

### Unresolved review count

| Field | Exact definition |
| --- | --- |
| `unresolvedReviewCount` | Count of `memoryId` where **any** of: (a) `resolvedAction == review`, or (b) orphan-merge safety downgrade applied, or (c) `conflictDetected == true` on the resolution |
| `conflictReviewCount` | Subset where `conflictDetected == true` |
| `suppressedActionCount` | `sum(len(resolution.suppressedActions) for resolution in memoryResolutions)` |
| `simulationWarningCount` | `len(simulationWarnings)` |

Orphan-merge downgraded removables are included in `unresolvedReviewCount` and `simulatedReviewQueue` even when the source resolution remains `merge`.

## Simulation mode comparability

| Mode | Included handlers | Metric comparability |
| --- | --- | --- |
| `full` | merge + archive + review/keep classification | Baseline mode for benchmarks |
| `merge_only` | merge + review/keep classification | Not comparable to `full`; `simulationMode` must be recorded |
| `archive_only` | archive + review/keep classification | Not comparable to `full`; `simulationMode` must be recorded |

Benchmark fixtures and regression guards must record `simulationMode`. Cross-mode metric comparison is invalid without explicit normalization.

## Summary rollups

`SimulationMetrics` aggregates the above into a single object suitable for JSON/Markdown reporting and simulation gold-fixture comparison. Published simulation benchmark artifacts must include a disclaimer that metrics are structural estimates.

# Safety Model

Simulation safety is enforced by architecture, not operator discipline.

## Guarantees

1. **Source immutability** — `AnalysisReport.memories` and input files are never written to. Handlers receive copies only.
2. **Copy-on-simulate** — `deep_copy` (or equivalent) creates new record instances before any transformation.
3. **No side effects** — no file I/O, no network I/O, no database writes in the simulation module.
4. **No elevation** — simulation cannot promote `review` to `merge` or `archive` without an explicit future workflow layer.
5. **Deterministic application** — fixed handler order and resolution-driven logic; no randomness, no model calls.
6. **Isolated output** — `SimulationReport` is a new object; it does not mutate `AnalysisReport`.
7. **Explicit warnings** — ambiguous cases produce `simulationWarnings` instead of silent removal.

## Failure modes and safe defaults

| Condition | Safe behavior |
| --- | --- |
| Missing resolution for a memory | Treat as `keep` (no change) |
| Merge removable without keeper | Orphan-merge safety: `ORPHAN_MERGE_NO_KEEPER` warning + no removal + review-queue accounting |
| Duplicate removal attempt | Warning + skip second removal |
| Empty memory store | Return zero deltas; no error |
| No resolutions | Simulated store equals source; metrics all zero delta |

## Boundary with execution

Simulation output is **informational**. It must be labeled as simulated in all report surfaces. No field in `SimulationReport` should be interpretable as an execution authorization.

# Explainability

Every simulated change must trace to V0.6 recommendation evidence. When a `Recommendation` record is absent, simulation uses the **normative fallback path** below.

## Primary trace chain

```
MemoryRecord.id
    → MemoryResolution (resolvedAction, role, recommendationId, confidence, suppressedActions)
        → Recommendation (reason, evidence[], affected_memories)   [when present]
            → GovernanceAction / lifecycle / trust sources (via evidence refs)
```

## Normative fallback path (M2)

V0.6 filters `report.recommendations` to records with evidence. Many memories — especially implicit `keep` resolutions (`recommendationId = rec:implicit:keep:{id}`) — have a `MemoryResolution` but **no** matching `Recommendation`.

When `recommendationId` is not found in the recommendation index:

1. **Do not** call `plan_recommendations()` or regenerate recommendations.
2. Build explainability from `MemoryResolution` fields:
   - `resolvedAction`, `role`, `confidence`, `conflictDetected`, `suppressedActions`
3. Enrich from `AnalysisReport.validation`:
   - `memoryLifecycle.memoryLifecycleAssignments[memoryId]` → lifecycle evidence
   - `memoryEvolutionAudit` cases referencing `memoryId` → evolution `caseId`
   - `categoryQuality` unknown/conflict samples → taxonomy evidence
4. Enrich from `AnalysisReport.clusters` / `actions` when `recommendationId` or resolution role implies cluster membership.
5. Set `explainabilitySource = "resolution_fallback"` on the simulated event.

### Implicit keep resolutions

For `recommendationId` matching `rec:implicit:keep:{memoryId}` or `rec:keep:{memoryId}`:

| Field | Fallback value |
| --- | --- |
| `reason` | `"Implicit retention: Active lifecycle with no remediation signal."` when lifecycle is Active; else `"Implicit retention: no competing remediation signal."` |
| `evidenceRefs` | `resolution:{recommendationId}`, plus `lifecycle_assignment:{memoryId}` when assignment exists |
| `explainabilitySource` | `resolution_fallback` |
| Simulated event | No structural change; may omit from `simulatedReviewQueue` unless flagged for display |

Implicit keeps do **not** require a `Recommendation` record for explainability completeness.

### Fallback reason synthesis (normative)

| `resolvedAction` | Fallback `reason` when `Recommendation` absent |
| --- | --- |
| `keep` | Implicit retention per lifecycle/absence (see above) |
| `review` | `"Review required per memory resolution."` + append lifecycle/taxonomy signal summary when present |
| `merge` | `"Merge resolution applied in simulation."` + cluster trust/policy refs from evidenceRefs |
| `archive` | `"Archive resolution applied in simulation."` + lifecycle state from assignment |

Simulation does **not** invent new natural-language narratives beyond these templates and existing evidence signal strings.

## evidenceRefs encoding (normative)

Canonical string format for `evidenceRefs` tuples:

| Prefix | Example | Source |
| --- | --- | --- |
| `resolution:` | `resolution:rec:implicit:keep:mem_1` | `MemoryResolution.recommendationId` |
| `recommendation:` | `recommendation:rec:merge:cluster-3` | `Recommendation.recommendationId` |
| `governance_action:` | `governance_action:merge_cluster:cluster-3` | `GovernanceAction.actionId` |
| `policy:` | `policy:balanced.approve-high-trust-safe-action` | `RecommendationEvidence.ruleId` |
| `lifecycle:` | `lifecycle:contradiction_mem1_mem2` | evolution/lifecycle `caseId` |
| `lifecycle_assignment:` | `lifecycle_assignment:mem_1` | assignment entry |
| `cluster_trust:` | `cluster_trust:cluster-3` | cluster ID |

Every simulated removal (`simulatedMerges`, `simulatedArchives`) must include at least one `evidenceRef`, produced via primary or fallback path.

## SimulatedExplainability block (proposed)

Each simulated event carries:

```
SimulatedExplainability:
  explainabilitySource: "recommendation" | "resolution_fallback"
  recommendationId: str
  reason: str
  evidenceRefs: tuple[str, ...]
  resolutionSnapshot: { resolvedAction, role, confidence, conflictDetected }  # optional JSON object
```

## Required explainability fields per simulated event

| Event type | Minimum trace fields |
| --- | --- |
| Merge removal | `memoryId`, `recommendationId`, `keeperId`, `trustLevel`, `evidenceRefs`, `explainabilitySource` |
| Archive removal | `memoryId`, `recommendationId`, `lifecycleState`, `evidenceRefs`, `explainabilitySource` |
| Review hold | `memoryId`, `recommendationId`, `reason`, `conflictDetected`, `suppressedActions`, `explainabilitySource` |
| Keep (implicit or explicit) | `memoryId`, `recommendationId`, `evidenceRefs`, `explainabilitySource` |

## Report surfaces

- **JSON:** structured `simulatedMerges`, `simulatedArchives`, `simulatedReviewQueue` with `explainabilitySource` and `evidenceRefs` on each entry.
- **Markdown:** "Simulation Summary" section with before/after structural estimates and explicit **"structural estimate — not re-analyzed"** labeling.
- **Terminal:** compact delta line + top warnings.

Human-readable `reason` strings are copied from `Recommendation.reason` when available; otherwise fallback templates apply.

# Output Contracts

The following schemas are **proposed** for V0.7 implementation. They are design artifacts only — not implemented in `memd/contracts.py` until a follow-up implementation phase.

## `SimulationMetrics` (proposed)

```python
class SimulationMetrics(FrozenModel):
    # Store size
    memoryCountBefore: int
    memoryCountAfter: int
    memoryCountDelta: int
    memoryReductionPercentage: float

    # Estimated structural duplicate / compression (NOT benchmark-equivalent)
    estimatedRemovableBefore: int
    estimatedRemovableAfter: int
    estimatedDuplicateReduction: int
    estimatedStructuralCompressionBefore: float
    estimatedStructuralCompressionAfter: float
    estimatedCompressionGain: float
    estimatedTrustedRemovableBefore: int
    estimatedTrustedRemovableAfter: int
    estimatedTrustedStructuralCompressionBefore: float
    estimatedTrustedStructuralCompressionAfter: float
    estimatedTrustedCompressionGain: float
```

## `SimulationReport` (proposed)

```python
class SimulatedExplainability(FrozenModel):
    explainabilitySource: str  # "recommendation" | "resolution_fallback"
    recommendationId: str
    reason: str = ""
    evidenceRefs: tuple[str, ...] = ()
    resolutionSnapshot: dict[str, Any] = Field(default_factory=dict)

class SimulatedMergeGroup(FrozenModel):
    recommendationId: str
    keeperId: str
    removedIds: tuple[str, ...]
    clusterId: str = ""
    trustLevel: str = ""
    trusted: bool = False
    explainability: SimulatedExplainability

class SimulatedArchiveEntry(FrozenModel):
    memoryId: str
    lifecycleState: str
    recommendationId: str
    archivedRecord: MemoryRecord
    explainability: SimulatedExplainability

class SimulatedReviewEntry(FrozenModel):
    memoryId: str
    recommendationId: str
    reason: str = ""
    conflictDetected: bool = False
    suppressedActions: tuple[str, ...] = ()
    explainability: SimulatedExplainability
    orphanMergeDowngrade: bool = False  # true when review queue entry from ORPHAN_MERGE_NO_KEEPER

class SimulationWarning(FrozenModel):
    code: str  # e.g. ORPHAN_MERGE_NO_KEEPER, DUPLICATE_REMOVAL_SKIPPED
    memoryId: str = ""
    message: str
    recommendationId: str = ""

class SimulationReport(FrozenModel):
    simulationId: str
    sourceMemoryCount: int
    simulatedMemoryCount: int
    simulatedMemories: tuple[MemoryRecord, ...]
    simulatedMerges: tuple[SimulatedMergeGroup, ...]
    simulatedArchives: tuple[SimulatedArchiveEntry, ...]
    simulatedReviewQueue: tuple[SimulatedReviewEntry, ...]
    simulationWarnings: tuple[SimulationWarning, ...]
    metrics: SimulationMetrics

    # Provenance
    analysisReportRef: str = ""
    policyProfile: str = ""
    simulationMode: str = "full"  # full | merge_only | archive_only
    metricsDisclaimer: str = "Structural estimates only; not benchmark-equivalent compression."
```

### `simulationId` determinism (normative)

`simulationId` is a stable hash of canonical inputs:

1. Sorted `memoryId` + `content` pairs from `AnalysisReport.memories`
2. Sorted `memoryResolutions` serialized as `(memoryId, resolvedAction, role, recommendationId, conflictDetected)`
3. `simulationMode`
4. `AnalysisReport.policySummary.profile` (string value)

Excludes non-deterministic fields (timestamps, object ids). Same inputs → same `simulationId`.

## Pipeline position (proposed)

```
analyze_file() → AnalysisReport
simulate_recommendations(report) → SimulationReport
render_simulation(report) → JSON / Markdown / terminal (future)
```

Simulation is a **separate step** after analysis. It does not modify `analyze_file()` behavior in V0.7 unless explicitly wired behind a flag in a later integration phase.

# Benchmark Strategy

Simulation quality is distinct from recommendation correctness (ADR-002). Benchmarks measure **whether simulated impact matches expected structural outcomes** given known recommendation inputs.

## What simulation benchmarks are not

- Not a repeat of recommendation accuracy testing
- Not end-to-end agent performance
- Not operator satisfaction
- Not proof that merges/archives should have been executed in production

## Proposed benchmark dimensions

### 1. Structural fidelity (gold fixture)

Create `tests/fixtures/simulation_gold.json` with cases specifying:

- Input memories + embedded or referenced resolutions
- Expected `memoryCountAfter`, `estimatedDuplicateReduction`, `archivedByLifecycleState`
- Expected merge groups (keeper + removed sets)
- Expected `unresolvedReviewCount` and `simulationMode`

**Metrics:** case-level exact match on structural outcomes; tolerance-based match on percentage fields. Gold fixtures validate **simulation structural fidelity**, not LongMemEval/PERMA/recommendation benchmark numbers.

### 2. Regression guards on published benchmarks

Run simulation on existing benchmark exports (LongMemEval cleaned, PERMA `user108`) in `simulationMode = full` and assert **structural guard inequalities** only:

- `estimatedTrustedCompressionGain <= estimatedCompressionGain` (trusted structural gain does not exceed headline structural gain)
- Review-dominant exports: `unresolvedReviewCount / memoryCountBefore` exceeds `resolutionsApplied / memoryCountBefore`
- **Conditional:** when `recommendationsWithStructuralEffect > 0` for merge recommendations, `estimatedDuplicateReduction > 0`

Guards are optional smoke checks. They do **not** assert equivalence to published LongMemEval or PERMA compression percentages.

### 3. Safety invariants (property tests)

| Property | Assertion |
| --- | --- |
| Source immutability | `AnalysisReport.memories` unchanged after simulation |
| Idempotency | Simulating twice yields identical `SimulationReport` |
| Monotonic reduction | `memoryCountAfter <= memoryCountBefore` when only merge/archive remove |
| Review preservation | Review IDs present in before and after active store |
| No orphan removal | Every removed ID has a corresponding log entry |

### 4. Explainability completeness

- Every simulated removal has non-empty `recommendationId` and at least one `evidenceRef` (primary or fallback)
- Implicit-keep paths satisfy completeness via `explainabilitySource = resolution_fallback`
- Warning count zero on gold fixture happy paths
- Orphan-merge fixtures assert `ORPHAN_MERGE_NO_KEEPER` warning + zero removals + review-queue entries with `orphanMergeDowngrade = true`

## Proposed benchmark artifacts

| Artifact | Purpose |
| --- | --- |
| `examples/benchmarks/simulation_evaluation.json` | Aggregated structural metrics |
| `examples/benchmarks/simulation_evaluation.md` | Human-readable summary |
| `scripts/run_simulation_evaluation.py` | Reproducible runner (future) |

## Progression gate (V0.7)

Before V0.8 workflow design begins:

1. Simulation module exists and passes gold-fixture structural tests
2. Safety invariants hold in CI
3. LongMemEval/PERMA regression guards documented with observed deltas
4. Simulation labeled as dry-run in all report outputs

Recommendation evaluation (ADR-002) remains a prerequisite; simulation benchmarks add a **second validation layer** for operational impact modeling.

## Glossary

| Term | Meaning |
| --- | --- |
| **Structural estimate** | Metric derived by adjusting pre-analysis cluster/resolution data against simulated active store IDs; not from re-running analyze or benchmark pipelines |
| **Active simulated store** | In-memory copy of memories after merge/archive removals; `S_after` |
| **Reference metric** | Pre-simulation `AnalysisReport.metrics` value cited for context only |
| **Orphan-merge safety** | Simulation-only guard when merge keeper is absent; no removal; `ORPHAN_MERGE_NO_KEEPER` warning |
| **Resolution fallback** | Explainability path using `MemoryResolution` + `validation` when `Recommendation` record is absent |
| **Recommendation outcome utilization** | Fraction of eligible merge/archive recommendation groups that produced structural simulated effects |
