# V0.8 Phase 2 Workflow Planner Audit

Date: 2026-06-20  
Auditor: Independent architecture review (read-only)  
Authority: ADR-004 (architectural), `V0.8-PHASE2-WORKFLOW-PLANNER-PLAN.md` (implementation)

---

## Executive Summary

Phase 2 delivers a functional, deterministic, read-only workflow planner in `memd/workflows.py` with 75 passing focused tests. The two-pass architecture, canonical identity pipeline, simulation verification (R2/R3/R4 split), cross-pass integrity aggregation, policy fail-closed missing-action handling, and ADR-004 aggregate precedence are implemented and largely conform to the accepted plan.

**No mutations, I/O, execution, or pipeline attachment were found.** Phase 1 contracts remain untouched. Validation commands pass for workflow scope; full suite retains one pre-existing unrelated failure.

Several **medium-severity plan divergences** were identified: `UNSUPPORTED_ACTION` is never emitted; conflicting-duplicate variant payloads are not fully represented in evidence; archive/merge memory overlap suppresses structural steps but does not route conflict review; lifecycle-mixed detection is partial; and global `DUPLICATE_REMOVAL_SKIPPED` warnings attach item blockers to every item. None of these break core safety of the read-only planner, but they should be hardened before treating Phase 2 as fully closed.

**Final verdict: APPROVED WITH LIMITATIONS (estimated 90% compliance).**

## Policy and Review Routing

### Policy aggregation (`_aggregate_policy`, L537–563)

- Precedence `blocked > requires_review > approved` with sorted `actionId` iteration — **order-independent**
- Missing referenced IDs → integrity path in Pass B — **conformant**
- **Gap:** linked actions with `policyDecision=None` are skipped silently; effective result may become `APPROVED` without all linked actions being resolved (see M3)

### Review routing

- Predicates use structured fields: simulation events, audit sets, evidence `source`, subtypes, policy enum — **not** blocker messages or step descriptions
- `general` only when independently required and no specific subtype (L718–736) — **conformant**
- Safe merge/archive with approved policy and no signals → no review — tested (T26/T27)
- Phase 1 cross-field validator satisfied: blocked/integrity items get `reviewRequirement.required=False` (L708–716, L987–995)
- **Gap:** `policy_blocked` subtype not emitted when policy is `BLOCKED` because `_match_specific_subtypes` returns early (L633–634); item is `BLOCKED` not review-routed — acceptable for blocked posture but diverges from ADR subtype table when review and block overlap
- **Gap:** `lifecycle_mixed` only from `subtype=="archive_merge_conflict"`, not from resolution `suppressedActions` (see M4)
