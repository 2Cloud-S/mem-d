## Fixes Applied

Fixes were strictly scoped to Phase 4 workflow evaluation files:

1. Updated `tests/fixtures/workflow_gold.json` to align semantic expectations
   with hardened audit requirements for:
   - scored review subtype coverage;
   - explicit planner-version disposition for `policy_blocked`;
   - a lifecycle-only scored review subtype case;
   - a scored simulation-safety route;
   - a non-policy non-overridable blocker decision rejection case.
2. Updated `memd/benchmarks/workflow_evaluation.py` to:
   - validate subtype non-vacuity from scored review-routing checkpoints only;
   - record evaluator-owned source report and decision input plan immutability
     snapshots;
   - block CI-observable network/process/write effects across all planning and
     decision-stage safety replays;
   - report H1-H7 gate IDs in readiness-boundary checkpoint failures.
3. Updated `tests/test_workflow_evaluation.py` to prove:
   - duplicate/collision fixtures are invalid before scoring;
   - metadata-only subtype coverage is insufficient;
   - source and decision input mutations fail safety checks;
   - Python-level writes are blocked by the no-effects guard;
   - H-gate failure messages name the relevant gate.

No workflow planner, contracts, pipeline, reports, recommendations, simulation,
policy, or actions files were changed.
