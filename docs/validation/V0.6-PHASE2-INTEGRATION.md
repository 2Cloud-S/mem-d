## Integration Points

### Pipeline (`memd/pipeline.py`)

```
… → plan_governance_actions → apply_policy → plan_recommendations → AnalysisReport
```

```python
recommendations, memory_resolutions, recommendation_summary = plan_recommendations(
    records,
    clusters,
    validation,
    insights,
    actions,
    metrics=metrics,
)
```

- Runs after policy annotations are available on governance actions.
- Reuses existing `metrics`, `validation`, `insights`, and `actions` outputs.
- Does not call `apply_policy` or re-plan governance actions.

### AnalysisReport extension

| Field | Type | Default |
| --- | --- | --- |
| `recommendations` | `tuple[Recommendation, ...]` | `()` |
| `memoryResolutions` | `tuple[MemoryResolution, ...]` | `()` |
| `recommendationSummary` | `RecommendationSummary` | empty summary |

Existing fields unchanged. Manual `AnalysisReport(...)` construction without recommendation fields remains valid.

### Reporting (`memd/reports.py`)

**JSON** (`report_to_dict`):

- `recommendations`
- `memoryResolutions`
- `recommendationSummary`

**Markdown** — new section after Action Plan:

- `## Recommendation Summary`
- `### Merge recommendations`
- `### Archive recommendations`
- `### Review recommendations`
- `### Keep recommendations` (only when keep recommendation records exist)

**Terminal**:

- Table: top 5 non-keep recommendations with action, confidence, reason, evidence summary
- Rollup line: merge / archive / review counts

Recommendations without evidence are excluded from display and from the emitted `recommendations` tuple.

## Evidence Improvements (Phase 1 audit)

| Audit finding | Phase 2 change |
| --- | --- |
| Compression metrics not in evidence | `compression_metrics_evidence()` added; attached to review recommendations via `with_compression_evidence()` when `metrics` or `validation.compressionDrivers` available |
| Conflict review evidence thin | `conflict_review_evidence()` bundles lifecycle, governance, cluster trust, and conflict resolution signals |
| Empty evidence recommendations | `recommendation_has_evidence()` filter before return |

No changes to precedence, conflict matrix, or confidence formulas.

## Limitations

1. **`include_keep` not exposed via CLI** — keep recommendation records omitted by default; memory resolutions may still resolve to `keep`.
2. **`insights` parameter still unused** in recommendation planning (deferred).
3. **No recommendation benchmarking** — out of scope for Phase 2.
4. **Benchmark baseline renderers** — not updated to include recommendation sections unless they call `render_markdown` / full report dict (workflow uses existing baseline builder).
5. **Terminal shows top 5** non-keep recommendations only.
