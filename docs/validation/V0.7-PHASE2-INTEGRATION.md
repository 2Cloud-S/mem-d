## Summary

Integrated the V0.7 simulation layer into the normal Mem-D analysis path. `analyze_file()` now runs `simulate_recommendations()` after recommendation generation and attaches the result to `AnalysisReport.simulationReport`. JSON, Markdown, and terminal outputs expose simulation data additively. Explainability fallback enrichment (audit M2 residual) and property tests from the Phase 1 audit are implemented.

Simulation remains read-only, dry-run, non-persistent, and non-executing.

## Pipeline Integration

```
analyze_file()
  → … insights, governance, policy …
  → plan_recommendations()
  → AnalysisReport (without simulation)
  → simulate_recommendations(report)
  → AnalysisReport.simulationReport attached
```

- Simulation consumes recommendation outputs only.
- No recommendation regeneration or upstream re-evaluation.
- Source `AnalysisReport` fields unchanged before attachment; simulation operates on read-only input.

## Reporting Additions

### JSON (`report_to_dict` / `render_json`)

Additive top-level keys when `simulationReport` is present:

- `simulationReport`
- `simulationMetrics`
- `simulatedMergeGroups`
- `simulatedArchiveEntries`
- `simulatedReviewEntries`
- `simulationWarnings`

Reports without `simulationReport` omit these keys (backward compatible).

### Markdown (`render_markdown`)

New section **# Simulation Summary** with:

- Projected Store Impact (before/after/delta)
- Recommendation Outcomes (merges, archives, reviews, keeps)
- Structural Estimates (`estimatedDuplicateReduction`, `estimatedCompressionGain`, `estimatedTrustedCompressionGain`)
- Warnings (or "No simulation warnings.")

Includes structural-estimate disclaimer.

### Terminal (`render_terminal`)

Concise block:

```
Simulation:
  Store Delta: ±N memories
  Merge Effects: N
  Archive Effects: N
  Review Queue: N
  Warnings: N
```

## Known Limitations

1. **`simulationMode` fixed to `full`** — no CLI flag for `merge_only` / `archive_only`.
2. **No dedicated `memd simulate` subcommand** — simulation runs on every `memd analyze`.
3. **No simulation benchmark runner** — out of Phase 2 scope.
4. **`analysisReportRef` not populated** — provenance linking deferred.
5. **Simulated memory list not included in JSON top-level** — available inside `simulationReport` nested structure only.
6. **Pre-existing `memories.txt` parser test failure** — unrelated fixture gap.
