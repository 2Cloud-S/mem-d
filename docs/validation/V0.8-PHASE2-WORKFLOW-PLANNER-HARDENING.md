## 2. M1 Disposition

**Finding:** `UNSUPPORTED_ACTION` was never emitted.

**Disposition:** Specification clarification, no production code path required.

The closed `RecommendationAction` enum (`merge`, `archive`, `review`, `keep`, `defer`) is fully handled by explicit planner branches. `UNSUPPORTED_ACTION` remains a reserved blocker code for future contract extensions. Module docstring documents that any new action value must add an explicit planner branch in the same change.

**Test:** `test_m1_all_recommendation_actions_have_explicit_planner_path` parametrizes all enum members and confirms planning succeeds without `UNSUPPORTED_ACTION`.

## 3. M2 Conflict Routing Fix

**Finding:** Archive and merge targeting the same memory suppressed structural steps but did not route conflict review.

**Fix:** `_apply_structural_overlap_conflicts` runs after item draft construction and before integrity finalization. It:

- Detects memories targeted by both archive and merge structural operations
- Sets `conflictDetected=True`, `skip_structural_step=True`
- Adds `ReviewSubtype.CONFLICT` and routes primary queue to `ReviewQueueId.CONFLICT`
- Sets `reviewRequirement.required=True` and `plannerItemStatus=REQUIRES_REVIEW` unless already blocked
- Withholds both conflicting structural steps
- Recomputes queue rank deterministically

**Tests:** `test_m2_structural_overlap_routes_both_items_to_conflict_review`, `test_m2_structural_overlap_unrelated_items_unaffected`, `test_m2_structural_overlap_order_independent`, extended `test_t42`.

## 4. M3 Policy Fix

**Finding:** Linked actions with `policyDecision=None` were skipped, allowing premature `APPROVED`.

**Fix:** Restrictive precedence in `_aggregate_policy`:

```text
blocked > requires_review > unresolved(None) > approved
```

Unresolved existing actions yield `REQUIRES_REVIEW`. Missing referenced action IDs still produce `INPUT_INTEGRITY` and blocked items.

**Tests:** `test_m3_*` suite (six scenarios plus order independence and missing-vs-unresolved distinction).

---

## 5. M4 Lifecycle Routing Fix

**Finding:** `lifecycle_mixed` was only detected from recommendation subtype, not resolution `suppressedActions`.

**Fix:** `_build_audit_sets` adds memory IDs to `lifecycle_mixed_memory_ids` when a resolution's `suppressedActions` contains both `merge` and `archive` enum values (no free-form text parsing).

**Tests:** `test_m4_lifecycle_mixed_from_suppressed_actions_positive`, `test_m4_lifecycle_mixed_from_suppressed_actions_negative`.

---

## 6. M5 Warning-Scope Fix

**Finding:** Global `DUPLICATE_REMOVAL_SKIPPED` warnings (empty `recommendationId`) attached blockers to every item.

**Fix:** `_process_duplicate_removal_warnings` scopes blockers deterministically:

| Scope | Behavior |
| --- | --- |
| `recommendationId` present | Blocker on matching recommendation item only |
| `memoryId` present, no rec ID | Blocker on items whose `affectedMemoryIds` contain that memory |
| Both absent | `duplicate_removal_unscoped:{digest}` integrity cause; plan-level `INPUT_INTEGRITY`; no indiscriminate item blockers |

**Tests:** `test_m5_duplicate_removal_warning_scoped_by_recommendation_id`, `test_m5_duplicate_removal_warning_scoped_by_memory_id`, `test_m5_duplicate_removal_unscoped_warning_integrity_blocked`.

## 7. M7 Evidence Fix

**Finding:** Conflicting duplicate variant payloads were not represented in evidence.

**Fix:** Without modifying Phase 1 contracts, conflicting variants are represented as stable references:

```text
duplicate_variant:{recommendationId}:{variantDigest}
```

Added to `WorkflowEvidence.validationRefs` (sorted, deduplicated). Plan-level integrity blocker message references variant digests when present. Canonical survivor selection remains lowest digest.

**Tests:** `test_m7_conflicting_duplicate_variants_in_validation_refs`, `test_m7_conflicting_duplicate_variant_coverage_order_independent`.

**Limitation:** Full structured variant payload persistence requires a future contract extension.
