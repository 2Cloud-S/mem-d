## 2. M1 Verification

Inspected `WorkflowItem.validate_structure` in `memd/contracts.py` (lines 574–585).

### Enforced (confirmed)

| Rule | Implementation | Test |
| --- | --- | --- |
| Review action requires review | `action == REVIEW` → `reviewRequirement.required` | `test_review_action_requires_review_requirement` |
| Human approval requires review | `requiresHumanApproval` → required review | `test_requires_human_approval_requires_review_requirement` |
| Planner `requires_review` requires review | `plannerItemStatus == REQUIRES_REVIEW` → required review | `test_planner_requires_review_status_requires_review_requirement` |
| Policy `requires_review` requires review | `policyDecision == REQUIRES_REVIEW` → required review | `test_policy_requires_review_requires_review_requirement` |
| Merge/archive may require review | No prohibition on non-review action + required review | `test_merge_action_with_valid_required_review_is_valid`, `test_archive_action_with_valid_required_review_is_valid` |
| Proposed merge without review signals valid | No false-positive review forcing | `test_proposed_merge_without_review_signals_is_valid` |
| Blocked without review signals valid | `BLOCKED` not in requires_review predicate | `test_blocked_item_without_review_requirement_is_valid` |
| Inconsistent inputs fail closed | `raise ValueError(...)` when predicate true and `required=False` | All four negative M1 tests use `pytest.raises(ValidationError)` |

### Not present (correct per ADR-004)

No rule of the form:

```text
non-review action → reviewRequirement.required must be false
```

The validator only requires review when explicit review signals are present; it never rejects merge/archive items merely for carrying a required review requirement.

### ADR alignment

Recommendation action and review requirement remain separate dimensions, consistent with ADR-004 §5 and hardening interpretation in `V0.8-PHASE1-CONTRACTS-HARDENING.md`.

**M1 status: VERIFIED**
