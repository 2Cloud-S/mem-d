## Validator Coverage

Implemented Phase 1 structural validators for:

- `ReviewRequirement` required/non-required shape rules
- `WorkflowBlocker` non-overridable code enforcement
- `WorkflowStepOperation` merge/archive/review/retain/handoff structural constraints
- `WorkflowStep` step type consistency
- `WorkflowItem` id presence, role uniqueness, simulation ref prefix/uniqueness
- `WorkflowSummary` defensive copy for mapping inputs
- `WorkflowPlan` uniqueness checks, gap-free step sequences, blocker ref resolution, initial-plan invariants, integrity-blocked requirement, workflow hash format

Planner algorithms were intentionally not implemented.

## Deviations

No semantic deviations from accepted ADR-004 and revised Phase 1 plan.

Notable implementation choices aligned with the revised plan:

- `ApprovalDecision.targetType` uses `Literal["workflow_item"]` only (no speculative step/plan targets).
- `WorkflowPlan.plannerVersion` validates non-empty string but does not hard-bind to a current version constant.
- `WorkflowItem.suppressedActions` uses `tuple[RecommendationAction, ...]` to match upstream authoritative type.
- `roles` uses immutable `tuple[MemoryRoleAssignment, ...]` instead of mutable `dict`.

If any discrepancy is discovered during independent audit, it should be recorded there.
