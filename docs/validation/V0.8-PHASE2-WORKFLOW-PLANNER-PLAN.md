## 1. Phase 2 Scope

### Objective

Implement a deterministic, read-only workflow planner that:

1. Consumes a completed `AnalysisReport`.
2. Produces an immutable initial `WorkflowPlan` via `plan_workflows()`.
3. Applies explicit operator decisions via a separate pure `apply_workflow_decisions()` transformation.
4. Never mutates source memories, upstream report data, or attached simulation semantics.
5. Never executes recommendations or workflow steps.

### In scope

| Deliverable | Purpose |
| --- | --- |
| `memd/workflows.py` | Planner constants, hashing helpers, `plan_workflows`, `apply_workflow_decisions`, `WorkflowDecisionError` |

| `docs/validation/V0.8-PHASE2-WORKFLOW-PLANNER-IMPLEMENTATION.md` | Post-implementation record |

#### Warning reference fallbacks (normative)

`SimulationWarning` does **not** guarantee `memoryId` or `recommendationId`. Reference construction must not assume either is non-empty.

| Condition | `simulationRefs` entry |
| --- | --- |
| `memoryId` non-empty | `warning:{code}:{memoryId}` |
| `memoryId` empty | `warning:{code}:{eventDigest}` where `eventDigest = sha256_hex(contract_dict(warning))` |

Index lookup for warnings:

1. Primary: `(code, memoryId, recommendationId)` exact tuple match in `simulationWarnings`.
2. Fallback when `memoryId` empty on warning: match by `(code, eventDigest)` where digest is over the full warning contract dict.
3. Item linkage when `recommendationId` empty: attach via explicit planner rules (e.g., orphan merge warning linked through `simulatedReviewQueue` entry with `orphanMergeDowngrade=True` and matching `recommendationId` on the review entry) — never invent a `memoryId` not present on the warning contract.

## 3. API Specifications

### 3.1 `plan_workflows`

```python
def plan_workflows(
    report: AnalysisReport,
    *,
    options: PlanningOptions | None = None,
) -> WorkflowPlan:
    ...
```

| Aspect | Specification |
| --- | --- |
| Default options | `PlanningOptions(includeKeep=False)` when `options is None` |
| Mutation | **Must not** mutate `report` or nested frozen objects |
| Output | Frozen initial `WorkflowPlan`; `plannerStatus=INITIAL`; all `operatorItemStatus=NONE`; `decisionsFingerprint=""` |
| Determinism | Identical inputs → byte-identical initial plan JSON |
| Aggregate on initial plan | Never `READY_FOR_EXECUTION` |
| Integrity failures | Emit `INPUT_INTEGRITY` + `integrity_blocked` in output (fail-closed in plan, not raised) |

### 3.2 `apply_workflow_decisions`

```python
def apply_workflow_decisions(
    plan: WorkflowPlan,
    decisions: tuple[ApprovalDecision, ...],
) -> WorkflowPlan:
    ...
```

| Aspect | Specification |
| --- | --- |
| Nature | Pure immutable transformation |
| Input plan | Not mutated |
| `workflowPlanId` | **Unchanged** |
| Steps | **Unchanged** — no add, remove, or reinterpretation of `WorkflowStep` or operations |
| Updates | `operatorItemStatus`, `aggregateStatus`, `summary.itemsByOperatorStatus`, `decisionsFingerprint`, `plannerStatus` when decisions applied |
| Empty `decisions` | Return **input `plan` unchanged** (same object reference) |

### 3.3 `WorkflowDecisionError`

```python
class WorkflowDecisionError(ValueError):
    """Deterministic rejection of an invalid decision batch."""
```

Raised only after full batch validation fails. Input `plan` unchanged.

### 3.4 Decision batch semantics (fail-closed, atomic)

| Case | Behavior |
| --- | --- |
| Empty decisions | Return input `plan` unchanged |
| Batch validation | Validate **entire** batch before any transformation |
| Duplicate identical decisions | Normalize to one; idempotent |
| Conflicting decisions for same `targetId` | `WorkflowDecisionError` |
| Unknown `targetId` | `WorkflowDecisionError` |
| Target `plannerItemStatus==BLOCKED` | `WorkflowDecisionError` |
| Non-overridable blocker in `blockerRefs` | `WorkflowDecisionError` |
| Decision on `keep` item | `WorkflowDecisionError` |
| Valid batch | Apply all; recompute operator summary + aggregate + fingerprint |
| Update previously decided item | Allowed — new decision overwrites `operatorItemStatus` for that item |
| Same effective decision reapplied | Idempotent; byte-identical output |

### 3.5 Multi-call decision semantics

`apply_workflow_decisions` is cumulative on operator state:

1. Call 1: approve item A → A approved; fingerprint encodes A.
2. Call 2 on decided plan: approve item B → A remains approved; fingerprint encodes A and B.
3. Repeat call 2 with identical batch → byte-identical output.

`rationale` and `decidedBy` are validation inputs only in V0.8. `WorkflowPlan` does not persist decision history; see §6.8.

### 3.6 Review approval semantics (ADR S21)

Operator `APPROVED` on a review item = acknowledged only. Does not create merge/archive intent or structural handoff eligibility.

## 5. Two-Pass Planner Algorithm

`plan_workflows()` uses two explicit passes and a mutable **`PlanningContext`** (§5.1) that accumulates integrity cause keys across both passes. **Pass B must not construct Pydantic contracts whose IDs depend on `workflowPlanId` before Pass A completes.**

### 5.1 `PlanningContext` (internal mutable state)

| Field | Purpose |
| --- | --- |
| `normalized_recommendations` | Sorted, deduplicated recommendation list |
| `integrity_cause_keys` | Sorted-unique cause keys appended in Pass A and Pass B |
| `conflicting_duplicate_variants` | Map `recommendationId` → sorted canonical variant digests (conflicts only) |
| `simulation_verification` | R2 result payload (§7.2) |
| `lookup_maps` | `memories_by_id`, `actions_by_id`, `categories_by_id`, `resolutions_by_rec`, simulation indexes |
| `sourceAnalysisRef` | R1 output |
| `workflowPlanId` | Fingerprint from `plannerSignalInputs` (§6.4) |
| `planningMode` | `FULL` or `RECOMMENDATIONS_ONLY` |
| `audit_sets` | Precomputed unknown/lifecycle/trust/conflict sets for routing |

**Integrity cause lifecycle (normative):**

1. Pass A appends causes: conflicting duplicate keys, R2 mismatch, R3 mismatch.
2. Pass B appends causes: missing `sourceActionId` mappings, R4 unresolvable `simulationRefs`.
3. **Do not** materialize the plan-level `INPUT_INTEGRITY` `WorkflowBlocker` until all integrity checks finish.
4. After final cause collection, compute aggregated plan-level `INPUT_INTEGRITY` `causeDigest` from sorted unique `integrity_cause_keys`.
5. Attach final `blockerId` to item drafts that reference plan-level integrity during finalization (steps B1/B2 may use a placeholder ref resolved in B1-finalize).
6. Only then: final `blockerRefs`, steps, summary, aggregate status, `WorkflowPlan` construction.

`workflowPlanId` is computed in Pass A from upstream inputs and **does not** require recomputation when Pass B discovers R4 causes — R4 affects blockers and aggregate status, not plan identity (simulation event data is already hashed).

### Pass A — Normalize, verify, fingerprint

| Step | Action | Complexity |
| --- | --- | --- |
| A1 | Validate inputs (`AnalysisReport` required; normalize `options`) | O(1) |
| A2 | Build lookup maps (`memories_by_id`, `actions_by_id`, `categories_by_id`, `resolutions_by_rec`) | O(n+m) |
| A3 | Build simulation indexes from `simulationReport` when present: `simulatedMerges` by `recommendationId`, `simulatedArchives` by `memoryId`, `simulatedReviewQueue` by `(memoryId, recommendationId)`, `simulationWarnings` by `(code, memoryId, recommendationId)` plus digest index for empty-`memoryId` warnings | O(s) |
| A4 | Precompute audit sets from `categories`, `validation`, `clusters`, `actions` (unknown, lifecycle alternate/mixed, trust, conflict) | O(n+m) |
| A5 | Normalize recommendations — sort by `(ACTION_PRIORITY_RANK[priority], recommendationId)`; deduplicate byte-equivalent; detect conflicting duplicates | O(n log n) |
| A6 | Conflicting duplicates — record `integrity_cause_keys` with variant digests (§6.3.1); byte-equivalent duplicates collapse without cause | O(n) |
| A7 | Verify simulation R2 and R3 — record integrity causes and `simulation_verification` payload (§7); **no R4 in Pass A** | O(n+m) |
| A8 | Compute `sourceAnalysisRef` — authoritative fingerprint from current report (R1); see §6.3 | O(n+m) |
| A9 | Compute `workflowPlanId` from `plannerSignalInputs` (§6.4) | O(n+s) |

Pass A output: `PlanningContext` with normalized recommendations, integrity cause keys (provisional), `sourceAnalysisRef`, `workflowPlanId`, `planningMode`, lookup maps, simulation indexes. **No `WorkflowItem`, `WorkflowBlocker`, or `WorkflowStep` instances yet.**

### Pass B — Materialize contracts

| Step | Action | Complexity |
| --- | --- | --- |
| B1 | Draft item-level blockers; resolve missing `sourceActionIds` → append integrity causes (§9.1.1); **do not** emit plan-level `INPUT_INTEGRITY` yet | O(n) |
| B2 | Create workflow item drafts → assign `workflowItemId`; build `ReviewRequirement`, roles, refs; assign provisional `simulationRefs` | O(n) |
| B3 | **R4** — resolve every `simulationRefs` entry against Pass A indexes; unresolvable → append integrity cause keys | O(n+s) |
| B4 | Finalize plan-level `INPUT_INTEGRITY` blocker from all `integrity_cause_keys`; assign `blockerId`; attach to affected item `blockerRefs` | O(c) |
| B5 | Instantiate `WorkflowItem` frozen models from drafts | O(n) |
| B6 | Create structured operations per item (draft dicts, not steps yet) | O(n+s) |
| B7 | Suppress structural steps for integrity-blocked items and items with R4 failures | O(n) |
| B8 | Create ordered step drafts → assign `stepId`, `sequence`, `dependsOnStepIds` (§9.4) | O(n log n) |
| B9 | Instantiate `WorkflowStep` frozen models | O(n) |
| B10 | Build `WorkflowEvidence` closure | O(n+s) |
| B11 | Calculate `WorkflowSummary` (one-pass) | O(n) |
| B12 | Compute `aggregateStatus` via `compute_aggregate_status(...)` (§11) | O(n) |
| B13 | Construct final immutable `WorkflowPlan` | O(1) |

### Draft vs contract rule

Use plain dataclasses or dict drafts in Pass B steps B1–B8. Instantiate Pydantic `FrozenModel` contracts only when all fields including IDs are known.

## 6. Identity Design

### 6.1 Canonical hashing algorithm (normative)

All digests use this exact pipeline:

```python
def canonical_json_bytes(value: object) -> bytes:
    return json.dumps(value, sort_keys=True, separators=(",", ":")).encode("utf-8")

def sha256_hex(value: object) -> str:
    return hashlib.sha256(canonical_json_bytes(value)).hexdigest()
```

**Contract serialization:**

```python
def contract_dict(model: BaseModel) -> dict:
    return model.model_dump(mode="json", exclude_none=False)
```

Rules:

1. Use `model_dump(mode="json", exclude_none=False)` for all upstream Pydantic contracts in hash payloads.
2. Preserve exact serialized float values; **do not round**.
3. Explicit `null` fields remain in JSON (`exclude_none=False`).
4. Enum values serialize as `.value` strings via `mode="json"`.
5. Sort unordered collections by documented stable keys before hashing.
6. Preserve semantically ordered collections (e.g. `WorkflowStep.sequence`) as-is when order is part of semantics.
7. `MemoryRecord.content` in fingerprints uses normalized content after contract validation (stripped `id` and `content`).
8. **Exclude** `AnalysisReport.workflowPlan` from all source fingerprints — hash a report projection with `workflowPlan=None`.
9. Never include timestamps, Python `hash()`, object identity, or list indexes.
10. Any safety-relevant upstream value change **must** change `workflowPlanId`.

**Sorted collection keys:**

| Collection | Sort key |
| --- | --- |
| `memories` | `id` |
| `recommendations` | `recommendationId` |
| `memoryResolutions` | `(memoryId, recommendationId)` |
| `actions` (policy subset) | `actionId` |
| `simulatedMerges` | `recommendationId` |
| `simulatedArchives` | `memoryId` |
| `simulatedReviewQueue` | `(memoryId, recommendationId)` |
| `simulationWarnings` | `(code, memoryId, recommendationId)` |
| `affectedMemoryIds` | lexicographic |
| `workflowItemIds` in step | lexicographic |
| `integrityCauseKeys` | lexicographic |
| `conflictingVariantDigests` per `recommendationId` | lexicographic |

**Duplicate recommendation equality:** byte-equivalent when `contract_dict(a) == contract_dict(b)` for normalized recommendations.

### 6.2 Identity dependency order

```
sourceAnalysisRef  →  workflowPlanId  →  workflowItemId / blockerId / stepId
```

### 6.3 `sourceAnalysisRef` (R1)

R1 **computes** the authoritative reference; it does not compare a value to itself.

```
report_for_hash = report.model_copy(update={"workflowPlan": None})
report_fingerprint = {
  "memories": sorted(
    [{"id": m.id, "contentFingerprint": sha256_hex(m.content)} for m in report_for_hash.memories],
    key=lambda x: x["id"],
  ),
  "memoryResolutions": sorted(
    [contract_dict(r) for r in report_for_hash.memoryResolutions],
    key=lambda r: (r["memoryId"], r["recommendationId"]),
  ),
  "recommendations": sorted(
    [contract_dict(r) for r in normalized_recommendations],
    key=lambda r: r["recommendationId"],
  ),
  "actions": sorted(
    [{"actionId": a.actionId, "actionType": a.actionType.value,
      "policyDecision": a.policyDecision.value if a.policyDecision else None,
      "policyRuleId": a.policyRuleId} for a in report_for_hash.actions],
    key=lambda a: a["actionId"],
  ),
  "policyProfile": report_for_hash.policySummary.profile.value,
}
sourceAnalysisRef = sha256_hex(report_fingerprint)
```

**External comparison (R3 only):** when `simulationReport.analysisReportRef` is non-empty, it must equal `sourceAnalysisRef`. Otherwise no self-comparison.

| Surface | Included | Rationale |
| --- | --- | --- |
| `categories` | Full `CategorizedMemory` tuples | Unknown-category routing |
| `validation` | Full `dict` | Lifecycle, evolution, category-quality signals (`memoryLifecycle`, `memoryEvolutionAudit`, `categoryQuality`, …) |
| `clusters` | Full `DuplicateCluster` tuples | Trust routing via `trustLevel`, `trustScore`, `trustReasons` |
| `actions` | Full `GovernanceAction` tuples | Policy, trust, `requiresHumanApproval`, evidence mapping |

| `simulationReport` projection | Event tuples only (§6.4.1) | R4 resolution indexes; warning/merge/archive/review predicates |
| `auditSets` | Derived sets | Not fully reconstructible from raw projections alone without re-implementing recommendation audit logic |
| `integrityCauseKeys` | Conflict duplicates only | Pass B integrity causes (R4, missing actions) affect blockers/status, not plan ID — simulation data already in projection |

| Excluded | Justification |
| --- | --- |
| `insights` | Not consumed by planner predicates |
| `metrics`, `actionSummary`, `recommendationSummary` | Derived reporting; no routing effect |
| `simulatedMemories`, `SimulationReport.metrics` | Not read (§2.1) |
| `workflowPlan` on report | Output slot; excluded per §6.1 rule 8 |

**Identity-change tests required** (§16): category→unknown, lifecycle signal change, policy decision change, trust signal change, simulation warning change.

#### 6.4.3 Conflicting-duplicate identity (`integrityCauseKeys`)

Byte-equivalent duplicate recommendations (identical `contract_dict`) **normalize away** without changing `workflowPlanId`.

Conflicting duplicates (same `recommendationId`, differing canonical dicts) **must** affect `workflowPlanId` via sorted `integrityCauseKeys` included in `plannerSignalInputs`.

For each conflicting `recommendationId`:

```
variants = sorted(
  [contract_dict(v) for v in all conflicting canonical recommendations],
  key=lambda d: sha256_hex(d),
)
variantBundleDigest = sha256_hex(variants)
integrityCauseKey = f"duplicate_conflict:{recommendationId}:{variantBundleDigest}"
```

| Property | Rule |
| --- | --- |
| Input order independence | Variants sorted by `sha256_hex(contract_dict)` before bundling |
| Different conflict variants → different plan IDs | Any canonical dict change alters `variantBundleDigest` |
| Identical multiplicity normalizes | Multiple byte-equivalent copies collapse to one variant in normalization |
| Evidence completeness | All conflicting variants listed in plan-level `INPUT_INTEGRITY` `WorkflowEvidence`; survivor selection for item drafting does not drop variant records from evidence |

### 6.5 `workflowItemId`

```
workflowItemId = sha256_hex({
  "workflowPlanId": workflowPlanId,
  "recommendationId": recommendation.recommendationId,
  "action": recommendation.action.value,
  "affectedMemoryIds": sorted(affected_memory_ids),
})
```

### 6.6 `blockerId` and collision prevention

Each blocker draft includes a **cause digest** distinguishing separate triggers:

```
causeDigest = sha256_hex({
  "code": code.value,
  "scope": "plan" | "item",
  "recommendationId": recommendation_id_or_null,
  "memoryId": memory_id_or_null,
  "sourceLayer": source_layer,
  "simulationRef": simulation_ref_or_null,
  "evidenceRef": evidence_ref_or_null,
  "integrityCause": integrity_cause_key_or_null,
})

blockerId = sha256_hex({
  "workflowPlanId": workflowPlanId,
  "causeDigest": causeDigest,
})
```

**Deduplication:** blockers with identical `causeDigest` collapse to one blocker; `blockerRefs` point to the single ID.

**Plan-level `INPUT_INTEGRITY`:** multiple integrity causes may aggregate into **one** plan-level blocker only when:

- `causeDigest` is computed from sorted unique `integrityCause` keys in the payload, and
- blocker `message` and `WorkflowEvidence` deterministically list every cause.

### 6.7 `stepId`

```
stepId = sha256_hex({
  "workflowPlanId": workflowPlanId,
  "stepType": step_type.value,
  "sequence": sequence,
  "workflowItemIds": sorted(workflow_item_ids),
  "operation": contract_dict(operation),
})
```

### 6.8 `decisionsFingerprint` (cumulative operator state)

`WorkflowPlan` does not store decision rationale or history. The fingerprint represents **cumulative resulting operator state** after all applied decisions:

```
decisionsFingerprint = sha256_hex(
  sorted(
    [
      {
        "workflowItemId": item.workflowItemId,
        "operatorItemStatus": item.operatorItemStatus.value,
      }
      for item in plan.items
    ],
    key=lambda row: row["workflowItemId"],
  )
)
```

| Plan state | Value |
| --- | --- |
| Initial (`plan_workflows`) | `""` |
| After any successful `apply_workflow_decisions` | SHA-256 of all emitted items |

Properties:

- Represents full resulting operator state, not latest input batch only.
- Repeated identical application → same fingerprint and byte-identical plan.
- Multi-call sequences preserve earlier item states in fingerprint.
- `rationale` / `decidedBy` affect validation only; not stored on plan (V0.8 limitation).
- `workflowPlanId` unchanged across decision application.

- **Do not** modify `simulation.py`.

| --- | --- | --- | --- |

Rules:

## 8. Review Routing

### 8.1 When review is required

`reviewRequirement.required = True` when **any** of:

- `action == REVIEW`
- `requiresHumanApproval == True`
- `plannerItemStatus == REQUIRES_REVIEW`
- effective `policyDecision == REQUIRES_REVIEW` (§9.1 aggregation)
- any **specific** subtype predicate matched (§8.2)

Otherwise `reviewRequirement.required = False` with empty subtypes, queues, and escalation signals.

### 8.2 Specific subtype predicates (ADR §5.12)

Evaluate in table order. **All** matches contribute to `subtypes` (sorted unique). **Do not** add `general` during predicate evaluation.

| Subtype | Structured predicate | Queue |
| --- | --- | --- |
| `orphan_merge_downgrade` | `SimulatedReviewEntry.orphanMergeDowngrade` or `warning:ORPHAN_MERGE_NO_KEEPER:…` on `simulationWarnings` | `review:simulation_safety` |
| `simulation_safety` | `SimulationWarning.code == DUPLICATE_REMOVAL_SKIPPED` | `review:simulation_safety` |
| `conflict` | resolution or recommendation `conflictDetected` or `subtype=="conflict"` | `review:conflict` |
| `policy_blocked` | aggregated effective policy `blocked` | `review:policy` |
| `policy` | aggregated `requires_review` or `subtype=="policy_blocked_merge"` | `review:policy` |
| `unknown_category` | unknown-category audit set or `category==Unknown` or `subtype=="unknown_category"` | `review:unknown_category` |
| `lifecycle_alternate` | non-empty `alternateLifecycleSignals` | `review:lifecycle` |
| `lifecycle_mixed` | `modifier=archive_merge_conflict` or suppressed merge+archive | `review:lifecycle` |
| `lifecycle` | evidence `source=="memory_lifecycle"` and not alternate/mixed | `review:lifecycle` |
| `low_trust` | `trustLevel==Low` or signal `trustLevel=Low` | `review:low_trust` |

### 8.3 `general` fallback (corrected)

```text
general applies only when review is independently required (§8.1)
AND no specific subtype matched (§8.2)
```

**Safe proposed merge/archive** (`requiresHumanApproval=False`, effective policy `approved`, `plannerItemStatus=proposed`, no conflict/safety signals):

- `reviewRequirement.required = False`
- **No** `general` subtype
- **No** review queues

**Review action with no specific subtype:**

- `required = True`; `subtypes = (GENERAL,)`; `primaryQueueId = review:general`

**Human-approval item with no specific subtype:**

- `required = True`; `subtypes = (GENERAL,)`

Primary queue precedence among matched subtypes: ADR §7.5 table. `queueRefs` = sorted matched queues.

#### 9.1.1 Policy decision aggregation

For `Recommendation.sourceActionIds`, resolve actions in **sorted `actionId` order** (order-independent):

```
precedence: blocked > requires_review > approved > missing
```

| Condition | Effective `policyDecision` |
| --- | --- |
| Any linked action `blocked` | `BLOCKED` |
| Else any linked action `requires_review` | `REQUIRES_REVIEW` |
| Else all linked actions resolved and `approved` | `APPROVED` |
| Any referenced `actionId` missing from `actions_by_id` | See missing-action rule below |

**Missing `sourceActionId` rule (deterministic, single path):**

When any `sourceActionId` on a recommendation is absent from `actions_by_id`:

1. Append `integrity_cause_keys` entry `f"missing_source_action:{actionId}"` for **each** missing ID (sorted in evidence).
2. Effective `policyDecision` for the affected item is `None`.
3. `plannerItemStatus` for the affected item is `BLOCKED`.
4. Affected item `blockerRefs` includes the aggregated plan-level `INPUT_INTEGRITY` blocker (assigned at Pass B finalization).
5. Plan aggregate becomes `INTEGRITY_BLOCKED` when any integrity cause exists.
6. **No** structural step is emitted for the affected item.
7. All missing action IDs appear in deterministic `WorkflowEvidence` on the plan-level `INPUT_INTEGRITY` blocker.

There is no alternate path (no “force review”, no partial policy inference).

**Tests must use reversed `sourceActionIds` order** to prove order independence for both aggregation and missing-ID evidence ordering.

#### 9.1.2 Planner item status

| Condition | `plannerItemStatus` |
| --- | --- |
| Effective policy `blocked` | `BLOCKED` |
| `action == DEFER` | `DEFERRED` |
| `reviewRequirement.required` | `REQUIRES_REVIEW` |
| Else | `PROPOSED` |

| `stepType` | Fields | Notes |
| --- | --- | --- |
| `REVIEW` | `reviewTargetIds` | From `simulatedReviewQueue` entries |
| `ARCHIVE` | `archiveTargetIds` | From `simulatedArchives` entries |

### 9.4 Step tiers, dependencies, and conflicts (normative)

**Tiers (global order):**

1. Review  
2. Archive  
3. Merge  
4. Retain (only if `includeKeep=True`)

Within tier: sort by `(queueRank, orderingKey, workflowItemId)`. Assign `sequence` 1..N gap-free after global ordering.

**Dependency rules:**

| Rule | Specification |
| --- | --- |
| Review before structural | Review steps precede archive/merge steps affecting the same `memoryId` |
| Structural depends on review | Archive/merge `dependsOnStepIds` includes review step IDs for overlapping `memoryId` targets |
| Archive+merge same memory | **Conflict** — withhold both structural steps; route to `review:conflict`; no archive→merge dependency |
| Blocked items | No structural steps |
| Review-required structural items | May retain proposed structural-intent step in plan, but step is **not handoff-eligible** until operator approval |
| `dependsOnStepIds` | Never create archive→merge dependencies for conflicting structural work |

**`apply_workflow_decisions`:** must not add, remove, or reinterpret steps or operations.
