## Summary

Implemented the V0.7 Simulation Layer as a strictly read-only dry-run system. `simulate_recommendations(report)` consumes `AnalysisReport` recommendation outputs and projects hypothetical store outcomes without mutating source memories, executing actions, or re-running any upstream layer.

Application order: merge removals → archive removals → review classification → keep (no-op).

## API

```python

```

Returns `SimulationReport` with simulated store, merge/archive/review artifacts, warnings, structural metrics, and explainability.

## Known Limitations

1. **Not wired into pipeline or CLI** — `simulate_recommendations()` is callable directly only.
2. **No report rendering** — JSON/Markdown/terminal output unchanged.
3. **Structural metrics only** — `estimatedCompressionGain` and related fields are not benchmark-equivalent; disclaimer on `SimulationReport.metricsDisclaimer`.
4. **`simulationMode` fixed to `full`** — `archive_only` / `merge_only` modes documented in architecture but not exposed as API parameters in Phase 1.
5. **No persistence** — simulation results are ephemeral in-memory objects.
6. **Gold fixtures use embedded validation** — cases build `AnalysisReport` via `plan_recommendations` helpers; not all cases run full `analyze_file` pipeline.
7. **No simulation benchmark runner** — gold fixture provides case-level checks only; no precision/recall rollup yet.
