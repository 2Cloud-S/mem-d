## 4. Immutability Decision

### Guarantee (explicit)

| Level | Guaranteed? | Mechanism |
| --- | --- | --- |
| Top-level model field reassignment | **Yes** | `FrozenModel` / `frozen=True` |
| Tuple element replacement | **Yes** | tuples are immutable |
| Nested `FrozenModel` fields | **Yes** | frozen child models |
| Nested `dict` values (if any remain on summary maps) | **No** | Pydantic frozen does **not** deep-freeze dict contents |

### Policy

1. **Do not claim deep immutability** for `dict[Enum, int]` summary maps (`itemsByAction`, `reviewQueueCounts`, etc.). Same posture as existing `AnalysisMetrics.categoryBreakdown`.
2. **Prefer tuples over dicts** on workflow item/plan fields where ADR allows — `roles` → `tuple[MemoryRoleAssignment, ...]`.
3. **Defensive copy on validation (optional):** `@field_validator` on summary dict fields may `return dict(value)` to avoid sharing caller-owned mutable dicts. Document as **copy-on-construct**, not deep freeze.
4. **Tests must document actual guarantee:**
   - M01: reassign top-level field → `ValidationError`
   - M02: mutate nested frozen `WorkflowItem` reference → `ValidationError`
   - M03: mutate `reviewQueueCounts` dict in place after construction → **may succeed**; test documents behavior explicitly (no false deep-immutability claim)
   - M04: `roles` tuple cannot be appended/replaced at field level

## 5. Revised Validators

### 5.1 `ReviewRequirement`

| `required` | Rule |
| --- | --- |
| **`True`** | `reason` strip + non-empty; `subtypes` non-empty; `queueRefs` non-empty; `primaryQueueId` not `None`; `primaryQueueId in queueRefs` |
| **`False`** | `subtypes == ()`; `queueRefs == ()`; `primaryQueueId is None`; `escalationSignals == ()`; `reason` may be `""` |

**Planner rule (Phase 2):** non-review items (`merge`, `archive`, `keep`, `defer` without review gate) must use `ReviewRequirement(required=False, ...)` with empty queue fields. Contract does not force non-review items into a queue.

### 5.2 `WorkflowStepOperation`

| `stepType` | Rules |
| --- | --- |
| **MERGE** | Exactly one non-empty `keeperId`; `len(removableIds) >= 1`; `keeperId not in removableIds`; `removableIds` unique; `archiveTargetIds` and `reviewTargetIds` empty |
| **ARCHIVE** | `len(archiveTargetIds) >= 1`; unique `archiveTargetIds`; merge/review fields empty |
| **REVIEW** | `len(reviewTargetIds) >= 1`; unique `reviewTargetIds`; merge/archive fields empty |
| **RETAIN / HANDOFF** | `keeperId`, `removableIds`, `archiveTargetIds`, `reviewTargetIds` all empty |

Helper: `_assert_unique_ids(ids: tuple[str, ...])` in validator module section of `contracts.py` (pure structural check, not planner logic).

### 5.3 `WorkflowPlan.plannerVersion`

| Rule | Owner |
| --- | --- |
| Non-empty stripped string | **Phase 1** `@field_validator` |
| Must equal current `WORKFLOW_PLANNER_VERSION` | **Phase 2 planner** emits; **not** schema validator |
| Reject unsupported version on consume | **Phase 2** planner / consumer logic |
| Deserialize historical `"1"`, `"2"`, … | **Allowed** by schema |

### 5.4 Other validators (unchanged intent)

- `WorkflowBlocker`: non-overridable codes per ADR §5.8
- `WorkflowPlan`: unique IDs, gap-free sequences, blocker ref resolution, INITIAL plan invariants (S19)
- `WorkflowItem`: `simulationRefs` prefix ∈ `SIMULATION_REF_PREFIXES`
