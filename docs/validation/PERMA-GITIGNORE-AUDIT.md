## Executive assessment

`datasets/perma/` contains benchmark source inputs (profile/task/evaluation data), plus a local cache subtree (`.cache/`) and OS noise (`.DS_Store`). Per active benchmark artifact policy, benchmark inputs should remain committable and cache/temp output should be ignored. Current `.gitignore` already contains a safe scoped ignore for `datasets/perma/.cache/` and globally ignores `.DS_Store`, so no additional ignore rules are required from this audit.

## Directory-by-directory purpose audit

### `profile/`

- **Purpose:** Canonical user preference/task metadata (directly consumed by `scripts/run_perma_benchmark.py`).
- **Source:** Upstream PERMA dataset source.
- **Reproducibility role:** **Required benchmark input** from a fresh clone.
- **Input vs output:** Input.

### `tasks/`

- **Purpose:** Native PERMA task instances and dialogue corpora.
- **Source:** Upstream PERMA dataset source.
- **Reproducibility role:** Input corpus backing the benchmark domain and future validation.
- **Input vs output:** Input.

### `evaluation/`

- **Purpose:** PERMA evaluation protocol metadata (`meta/overall/*.json` etc.).
- **Source:** Upstream PERMA dataset source.
- **Reproducibility role:** Input/contextual benchmark asset; not Mem-D generated output.
- **Input vs output:** Input.

### `WildChat-1M/`

- **Purpose:** Source slices referenced in PERMA dataset README.
- **Source:** Upstream PERMA package component.
- **Reproducibility role:** Part of source bundle; should not be hidden by ignore rules.
- **Input vs output:** Input.

### `.cache/`

- **Purpose:** Local HuggingFace cache and metadata mirrors.
- **Source:** Tooling-generated cache.
- **Reproducibility role:** Not required from clone; regenerated automatically.
- **Input vs output:** Output (cache/temp).

### Generated exports / benchmark outputs under `datasets/perma/`

- No canonical Mem-D benchmark outputs were found under `datasets/perma/`.
- Mem-D benchmark outputs belong under `examples/benchmarks/` (per workflow and policy), not under dataset source directories.

## Proposed Gitignore Changes

### Proposed changes

No additional `.gitignore` changes are required from this audit.

### Why this is safe

- `datasets/perma/.cache/` is already ignored (correct for cache/temp class).
- Global `.DS_Store` ignore already covers OS temporary files.
- No additional generated artifact patterns were found inside `datasets/perma/` that need ignore coverage.

### Why this does not hide benchmark inputs

- No dataset-wide ignore rule is applied to `datasets/perma/**`.
- Source input directories (`profile/`, `tasks/`, `evaluation/`, `WildChat-1M/`) remain committable.

### Policy compliance

- Aligns with `BENCHMARK-ARTIFACT-POLICY.md` classification: `datasets/perma/**` as Source Asset, cache/temp as ignored.

## Risk Review

### Dangerous ignore rule categories (checked)

| Risk type | Status | Notes |
| --- | --- | --- |
| Broad wildcard hiding inputs | Watchlist | Global `*.jsonl` is broad, but current `datasets/perma/` uses `.json`, not `.jsonl`; no immediate PERMA input loss. |
| Dataset-wide PERMA ignore | Not present | Prior dangerous `perma` rule has already been removed in current `.gitignore`. |
| Rule that hides benchmark inputs | Not found for PERMA | Source trees remain unignored except `.cache/`. |
| Rule that breaks clone reproducibility | Not found for PERMA inputs | Required runner inputs (`profile/<user>/profile.json`, `profile/<user>/tasks.json`) are not ignored. |

### Residual caution

- The global `*.jsonl` rule is still high-impact repository-wide and can hide future JSONL fixtures/inputs in other areas. This is not a PERMA-specific blocker in the current dataset structure, but remains a general maintenance risk.

## Audit conclusion

- Benchmark reproducibility for PERMA **is preserved** with current ignore behavior.
- No benchmark inputs in `datasets/perma/` are hidden by current `.gitignore`.
- Only cache/temp material (`datasets/perma/.cache/`, `.DS_Store`) is ignored.
- No `.gitignore` edits were applied as part of this audit because no additional ignore requirements were identified.
